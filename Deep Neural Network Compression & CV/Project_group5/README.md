# miniVGG CIFAR-10 Compression Pipeline

A 3-stage DNN compression pipeline applied to a VGG-style CNN trained on CIFAR-10.  
The pipeline compresses the model through **Pruning → Quantization → Huffman Coding**, achieving significant size reduction with minimal accuracy loss.

---

## How to Run

### 1. Install dependencies
```bash
pip install torch torchvision scikit-learn numpy
```

### 2. (optional) Place your trained weights (Already there if you don't want to retrain the model)
Copy your trained weights file into the `compressed_models/` folder and make sure `config.py` points to it:
```python
# config.py
WEIGHTS_PATH = 'compressed_models/final_model_weights.pth'
```

### 3. (optional) Run the compression pipeline 
(compression will take a lot of time sugested to directly go to next step, compressed files already present for convenience)
This loads the trained weights and runs all 3 stages (Prune → Quantize → Huffman), saving the compressed archive:
```bash
python main.py --mode compress
```
Outputs saved to `compressed_models/`:
- `pruned_model_weights.pth` — after stage 1
- `quantized_model.pth` — after stage 2 (also needed for BN/bias on reload)
- `huffman_encoded.pkl` — final compressed archive
- `compression_results.json` — summary metrics

### 4. Evaluate the compressed model
Loads the archive and reports accuracy without re-running compression:
```bash
python main.py --mode evaluate
```

## Project Structure

```
Project/
├── compressed_models/          # Output files from the pipeline
│   ├── final_model_weights.pth     ← place your trained weights here
│   ├── pruned_model_weights.pth    ← generated by compress mode
│   ├── quantized_model.pth         ← generated by compress mode
│   ├── huffman_encoded.pkl         ← final compressed archive
│   └── compression_results.json    ← metrics summary
│
├── models/
│   └── model_cifar.py          # miniVGG architecture definition
│
├── data/
│   └── data_loader.py          # CIFAR-10 dataset + dataloader setup
│
├── compression/
│   ├── prune.py                # L1 unstructured pruning (90% sparsity)
│   ├── quantization.py         # K-Means weight quantization
│   └── huffman.py              # Huffman encoding and decoding
│
├── utils/
│   ├── training.py             # Training loop and fine-tuning loop
│   ├── test_eval.py            # Accuracy evaluation and compression report
│   └── loading.py              # Load model from .pth or compressed archive
│
├── config.py                   # All hyperparameters and file paths
└── main.py                     # Entry point (--mode compress / evaluate)
```

## Results

| Stage | Accuracy | Size (KB) | Ratio |
|---|---|---|---|
| Original (baseline) | 89.45% | 4172.54 | 1.00x |
| After L1 Pruning (90%) | 85.60% | (sparse) | — |
| After K-Means Quantization | 85.63% | 102.16 | 40.71x |
| After Huffman Coding | 85.63% | 516.39 | 8.08x |

**Total accuracy drop : 3.82%**  
**Avg bits per weight : 7.860**  
**Huffman extra gain  : 1.2296x**
